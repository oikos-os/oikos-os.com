◈ Architecture

Not a wrapper. Not a chatbot skin. A full cognitive operating system.

⌂
Rooms
Isolated environments for different contexts. Each Room has its own system prompt, tools, memory scope, and personality. Switch with one command.
◈
Vault
Structured knowledge persistence. Markdown files organized by identity, knowledge, and patterns. Hybrid search (semantic + keyword). Your AI remembers.
⚡
42 MCP Tools
File operations, web browsing, research queues, git integration, system exec, notifications, vault management, inference routing — all built in.
🛡
Adversarial Security
Content classifier catches prompt injection, credential leaks, and identity override attempts. Output filtering. Audit trail. 10/10 gauntlet score.
🧠
Tiered Memory
Identity tier is always loaded. Knowledge tier retrieved per-query. The AI knows who it is before it knows what you asked. No role fragmentation.
🔌
Model Agnostic
Ollama, Claude, Gemini, OpenAI — switch providers without changing anything. Multi-provider inference routing. Use local models for privacy, cloud for power.

◈ Extend It

Add tools with a single decorator. No boilerplate.

# Register a custom MCP tool
from oikos.mcp import oikos_tool

@oikos_tool(scope="room")
def search_vault(query: str) -> list:
    """Search the knowledge vault."""
    return self.vault.hybrid_search(query)

◈ Philosophy

"Intelligence is cheap. Context is expensive. Build for context."

oikOS exists because AI that forgets you between sessions isn't intelligent — it's a stateless API call with a nice interface. Real intelligence requires memory, identity, and the ability to accumulate context over time.

The cloud is a loan. Your data on someone else's server is their leverage. oikOS puts the entire stack on your hardware — your vault, your models, your rules. No telemetry. No training on your conversations. No terms of service that change while you sleep.

"The cloud is a loan. Your memory outlives the model."
